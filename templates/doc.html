Company API Documentation

Welcome to the API documentation for the Company API. Below you will find all the available endpoints and their details.

API Endpoints

Endpoint Method Description
/api/v1/companies/ GET Get a list of all companies.
/api/v1/companies/{id}/ GET Get detailed information about a specific company.
/api/v1/companies/{id}/employees/ GET Get a list of employees for a specific company.
/api/v1/employees/ GET Get a list of all employees.
/api/v1/employees/{id}/ GET Get detailed information about a specific employee.
/api/v1/companies/ POST Create a new company.
/api/v1/companies/{id}/ PUT Update details of a specific company.
/api/v1/companies/{id}/ DELETE Delete a specific company.
/api/v1/employees/ POST Create a new employee.
/api/v1/employees/{id}/ PUT Update details of a specific employee.
/api/v1/employees/{id}/ DELETE Delete a specific employee.

Response Formats

All responses are returned in JSON format. For example, when retrieving details of a company or employee, the response will look like this:

{
    "id": 1,
    "name": "Company Name",
    "address": "123 Company St",
    "employees": [
        {
            "id": 1,
            "name": "John Doe",
            "position": "Manager"
        }
    ]
}

Notes

  • Replace {id} with the specific company or employee ID when accessing detailed information or making updates.
  • All endpoints require proper authentication, typically via an API key or token (ensure you include this in the headers).
